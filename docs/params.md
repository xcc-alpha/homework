# 参数选择与阈值设定说明 (Parameter Selection and Justifications)

本项目的分析阈值在 `config/query.yaml` 中统一管理。以下是针对光子计算（2015–2026年）文献计量网络分析的关键参数选择理由：

## 1. 最小边权重 (min_edge_weight = 2)
- 选择值： `2`
- 理由： 
  - 在共被引、关键词共现和合作网络中，权重为 `1` 的边（即仅共现或共引一次）通常包含大量的偶然关联和噪声。例如，两个毫不相关的词在同一篇综述中作为背景被提及，或者两个无关作者因为临时合作一篇短文而相连。
  - 将最小权重设为 `2`，可以有效过滤这些低频偶然关联，仅保留那些至少在两篇不同论文中共同出现的实质性关联，从而提高网络拓扑的信噪比和社团划分的稳定性。

## 2. 边数量上限 (top_edges = 200)
- 选择值： `200`
- 理由：
  - 如果不对保留的边数量进行限制，共被引或文献耦合网络会因为包含数千条边而变成密不透风的“毛线团”（Hairball），丧失可视化的可读性，并且会导致计算开销过大。
  - 保留权重排名前 200 的边，可以在展现网络的核心骨架和宏观社团划分（如区分“神经网络加速器”与“硅光混沌 reservoir computing”）的同时，确保图表视觉清晰、主次分明。

## 3. 标注节点数量 (top_labels = 15)
- 选择值： `15`
- 理由：
  - 在静态网络图（PNG）中，如果标注所有节点，会导致文字严重重叠、无法识别。
  - 本项目基于中介中心性（Betweenness Centrality）或度中心性（Degree Centrality），仅在图中标注排名前 15 的核心节点。这些节点通常是各社团的桥接点或核心代表，不仅起到标注骨干结构的作用，也避免了视觉混乱。

## 4. 布局随机种子 (layout_seed = 42)
- 选择值： `42`
- 理由：
  - NetworkX 的力导向布局（spring_layout）具有随机性。固定 `layout_seed` 可以确保每次运行代码生成的网络 PNG 空间分布完全一致，满足学术论文“完全可复现”的高标准要求。
